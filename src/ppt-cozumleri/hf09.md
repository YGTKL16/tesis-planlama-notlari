# HF09 — Yerleşim Tasarımı III — PowerPoint Çözümleri

## X. PowerPoint Çözümlü Örnek Sorular

### Örnek Soru-1 — MCRAFT: Yerleşim Vektörü ve Başlangıç Yerleşimi

!!! example "Problem"
    Aşağıdaki 5 bölüme sahip tesisin ölçüleri verilmektedir. Bina boyutu **20 m × 9 m**'dir. MCRAFT yöntemi için **3 bant** kullanarak yerleşim vektörünü belirleyiniz ve başlangıç yerleşimini oluşturunuz.

    | Bölüm | Alan (m²) |
    |---|---|
    | A | 30 |
    | B | 45 |
    | C | 51 |
    | D | 39 |
    | E | 15 |
    | Toplam | 180 |

**Çözüm — Yerleşim Vektörü:**

Alan büyüklüğüne göre küçükten büyüğe sıralama:
E(15) < A(30) < D(39) < B(45) < C(51)

Ancak MCRAFT vektörü BÜYÜKTEN KÜÇÜĞE sıralar (büyük bölümler önce yerleşir):
**Yerleşim vektörü: 1-3-4-2-5 (A-C-D-B-E)**

!!! info "Yerleşim Vektörü Nedir?"
    MCRAFT'ta bölümler numaralandırılır ve büyük alandan küçüğe doğru bir vektör oluşturulur. Bu vektör, algoritmanın başlangıç yerleşimini otomatik oluşturmasında kullanılır.

**3 Bant ile Başlangıç Yerleşimi:**

Bina 20m × 9m = 180 m². Her bant yüksekliği 9/3 = 3 m, genişliği 20 m:

```
Band 1 (en üst): C(51m²) → genişlik=51/3=17m, A(30m²) → 30/3=10m... 
```

Gerçek boyutlar bantlara sığdırılacak biçimde ayarlanır. Algoritma bu başlangıç yerleşiminden CRAFT mantığıyla ikili değiştirmeler yapar.

### Örnek Soru-2 — BLOCPLAN: Etkinlik Oranı ve REL-DIST Puanı (Tam Sayısal Çözüm)

!!! example "Problem"
    Aşağıdaki 5 bölümlü sistemde ilişki şeması verilmiştir (A=10, E=5, I=2, O=1, U=0, X=−10):

    | | D1 | D2 | D3 | D4 | D5 |
    |---|---|---|---|---|---|
    | D1 | — | A | U | E | U |
    | D2 | | — | U | I | I |
    | D3 | | | — | U | I |
    | D4 | | | | — | A |
    | D5 | | | | | — |

    **Önerilen yerleşim** (2×3 ızgara, sol-üst köşeden):

    ```
    | D1 | D4 | D5 |
    | D2 | D3 |    |
    ```

    Koordinatlar: D1=(0,0), D4=(1,0), D5=(2,0), D2=(0,1), D3=(1,1)

**Adım 1 — Çift İlişki Tablosu:**

İlişki ağırlıkları: A=10, E=5, I=2, O=1, U=0, X=−10

| Çift | İlişki | Değer |
|---|---|---|
| (1,2) | A | 10 |
| (1,3) | U | 0 |
| (1,4) | E | 5 |
| (1,5) | U | 0 |
| (2,3) | U | 0 |
| (2,4) | I | 2 |
| (2,5) | I | 2 |
| (3,4) | U | 0 |
| (3,5) | I | 2 |
| (4,5) | A | 10 |
| **Toplam (pozitif)** | | **31** |

**Adım 2 — Manhattan Uzaklıkları:**

$d_{ij} = |x_i - x_j| + |y_i - y_j|$

| Çift | Hesap | Uzaklık |
|---|---|---|
| D1–D2 | \|0−0\|+\|0−1\| | **1** (komşu) |
| D1–D3 | \|0−1\|+\|0−1\| | 2 |
| D1–D4 | \|0−1\|+\|0−0\| | **1** (komşu) |
| D1–D5 | \|0−2\|+\|0−0\| | 2 |
| D2–D3 | \|0−1\|+\|1−1\| | **1** (komşu) |
| D2–D4 | \|0−1\|+\|1−0\| | 2 |
| D2–D5 | \|0−2\|+\|1−0\| | 3 |
| D3–D4 | \|1−1\|+\|1−0\| | **1** (komşu) |
| D3–D5 | \|1−2\|+\|1−0\| | 2 |
| D4–D5 | \|1−2\|+\|0−0\| | **1** (komşu) |

**Adım 3 — REL-DIST Skoru:**

$$\text{REL-DIST} = \sum_{\text{çiftler}} \text{REL}_{ij} \times d_{ij}$$

| Çift | REL | Uzaklık | REL×d |
|---|---|---|---|
| (1,2) | 10 | 1 | 10 |
| (1,3) | 0 | 2 | 0 |
| (1,4) | 5 | 1 | 5 |
| (1,5) | 0 | 2 | 0 |
| (2,3) | 0 | 1 | 0 |
| (2,4) | 2 | 2 | 4 |
| (2,5) | 2 | 3 | 6 |
| (3,4) | 0 | 1 | 0 |
| (3,5) | 2 | 2 | 4 |
| (4,5) | 10 | 1 | 10 |
| **Toplam** | | | **39** |

**Adım 4 — Komşuluk Puanı $Z_A$ ve Etkinlik Oranı $ER$:**

Komşu çiftler (d=1): (1,2), (1,4), (2,3), (3,4), (4,5)

$$Z_A = \text{REL}_{12} + \text{REL}_{14} + \text{REL}_{23} + \text{REL}_{34} + \text{REL}_{45} = 10 + 5 + 0 + 0 + 10 = \mathbf{25}$$

Tüm pozitif ilişki toplamı ($Z_{max}$) = 31

$$ER = \frac{Z_A}{Z_{max}} = \frac{25}{31} = \mathbf{0.81} \approx \mathbf{\%81}$$

!!! success "Sonuç"
    - **REL-DIST = 39** → Düşük REL-DIST daha iyi (yüksek ilişkili bölümler birbirine yakın)
    - **Z_A = 25** → Komşuluk puanı
    - **ER = 0.81 (%81)** → Yerleşim oldukça iyi (kritik ilişkiler komşulukta)

    En kritik ilişkiler: (1,2)=A ve (4,5)=A → **her ikisi de komşu** ✓
    (2,5)=I ama d=3 (uzak) → potansiyel iyileştirme alanı

!!! tip "Sınavda Pratik İpuçları"
    - **$Z_A$ = komşu çiftlerin REL toplamı** (d=1 olanlar)
    - **$Z_{max}$ = tüm pozitif REL toplamı** (X değerleri hariç)
    - $ER = Z_A / Z_{max}$ → ne kadar yüksekse yerleşim o kadar iyi
    - **REL-DIST minimize edilir** (CRAFT'ta akış-maliyet çarpımı gibi)
    - Yüksek REL'li çiftler komşu olmalı, uzak olursa hem Z_A düşer hem REL-DIST artar
