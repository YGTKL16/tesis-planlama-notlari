# HF07 — Yerleşim Tasarımı I — PowerPoint Çözümleri

## X. PowerPoint Çözümlü Örnek Sorular — Süreç Yerleşimi

### Örnek Soru-1 — Malzeme Taşıma Maliyeti Minimizasyonu (6 Bölüm)

!!! example "Problem"
    60 × 40 ft'lik bir fabrikada **6 bölüm** (her biri 20×20 ft) malzeme taşıma maliyetini minimize edecek biçimde yerleştirilecektir.
    - **Komşu bölümler arası** taşıma maliyeti: **1 $/sefer**
    - **Komşu olmayan bölümler arası** taşıma maliyeti: **2 $/sefer**
    - 6! = 720 farklı olası çözüm

    Bölümler: 1-Assembly (Montaj), 2-Painting (Boya), 3-Machine Shop (Makine), 4-Receiving (Alım), 5-Shipping (Sevkiyat), 6-Testing (Test)

    Bölümler arası seferler (from-to matrix'ten):
    $f(1,2)=50$, $f(1,3)=100$, $f(1,6)=20$, $f(2,3)=30$, $f(2,4)=25$, $f(2,5)=10$, $f(3,4)=20$, $f(3,6)=100$, $f(4,5)=50$

**Başlangıç Yerleşimi:**

```
┌─────────────┬─────────────┬─────────────┐
│  1-Assembly  │  2-Painting  │  3-Machine   │
│     (A)      │     (B)      │    Shop (C)  │
├─────────────┼─────────────┼─────────────┤
│ 4-Receiving  │  5-Shipping  │  6-Testing   │
│     (D)      │     (E)      │     (F)      │
└─────────────┴─────────────┴─────────────┘
```

Komşu çiftler: (1,2), (1,4), (2,3), (2,5), (3,6), (4,5), (5,6)

**Başlangıç Maliyet Hesabı (Toplam = $570):**

| Çift | Sefer | Komşu? | Maliyet/sefer | Toplam |
|---|---|---|---|---|
| (1,2) | 50 | Evet | $1 | $50 |
| (1,3) | 100 | Hayır | $2 | **$200** |
| (1,6) | 20 | Hayır | $2 | $40 |
| (2,3) | 30 | Evet | $1 | $30 |
| (2,4) | 25 | Hayır | $2 | $50 |
| (2,5) | 10 | Evet | $1 | $10 |
| (3,4) | 20 | Hayır | $2 | $40 |
| (3,6) | 100 | Evet | $1 | **$100** |
| (4,5) | 50 | Evet | $1 | $50 |
| **TOPLAM** | | | | **$570** |

**İyileştirilmiş Yerleşim:**

3-Machine Shop'u 1-Assembly ile komşu yap; 6-Testing'i de 1 ile komşu yap:

```
┌─────────────┬─────────────┬─────────────┐
│  3-Machine   │  1-Assembly  │  2-Painting  │
│    Shop (A)  │     (B)      │     (C)      │
├─────────────┼─────────────┼─────────────┤
│  6-Testing   │  4-Receiving │  5-Shipping  │
│     (D)      │     (E)      │     (F)      │
└─────────────┴─────────────┴─────────────┘
```

Yeni komşu çiftler: (3,1), (1,2), (3,6), (1,4), (2,5), (6,4), (4,5)

**İyileştirilmiş Maliyet Hesabı (Toplam = $480):**

| Çift | Sefer | Komşu? | Maliyet/sefer | Toplam |
|---|---|---|---|---|
| (1,2) | 50 | Evet | $1 | $50 |
| **(1,3)** | 100 | **Evet** | $1 | **$100** ↓ |
| **(1,6)** | 20 | **Evet** | $1 | **$20** ↓ |
| **(2,3)** | 30 | **Hayır** | $2 | **$60** ↑ |
| (2,4) | 25 | Hayır | $2 | $50 |
| (2,5) | 10 | Evet | $1 | $10 |
| (3,4) | 20 | Hayır | $2 | $40 |
| (3,6) | 100 | Evet | $1 | $100 |
| (4,5) | 50 | Evet | $1 | $50 |
| **TOPLAM** | | | | **$480** |

!!! success "Sonuç"
    - Başlangıç maliyeti: **$570**
    - İyileştirilmiş maliyet: **$480**
    - **İyileşme: $90 (%15.8)** — yalnızca 3 bölümün yeri değiştirilerek

!!! tip "Strateji"
    Yüksek akışlı çiftleri (1,3)=100 ve (3,6)=100 komşu yap → en büyük tasarruf. 
    Bu manuel yaklaşım; CRAFT algoritması bu işlemi sistematik ikili değiştirmelerle (pairwise swap) yapar.
