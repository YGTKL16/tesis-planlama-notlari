## 1. Neden Hücresel İmalat? (5 Yaşındaki)

Sınıfta çalışıyorsun. Her öğrenci farklı konuyu farklı sırayla okuyor. Herkes herkesten kitap istiyor — kaos.

**Çözüm:** Benzer konuları okuyan öğrencileri aynı masaya oturt. Artık yalnız kendi masandaki arkadaşınla kitap paylaşıyorsun → daha az taşıma, daha az bekleme.

Fabrikada da aynı: benzer işlem gören parçaları ve o parçaları işleyen makineleri **hücre** adı verilen mini-fabrikalara topla.

**Hücresel imalatın faydaları:**
- **Azaltır**: Taşıma, WIP stoku, hareket
- **Basitleştirir**: Çizelgeleme, iletişim
- **İyileştirir**: Kalite, esneklik, verimlilik

## 2. Parça-Makine Matrisi

$$a_{ij} = \begin{cases} 1 & \text{parça } i \text{ makine } j \text{ üzerinde işlem görür} \\ 0 & \text{aksi halde} \end{cases}$$

Hedef: 1'leri köşegen boyunca bloklar halinde toplamak → her blok = bir hücre.

| | P1 | P2 | P3 | P4 | P5 | P6 |
|---|:-:|:-:|:-:|:-:|:-:|:-:|
| **M1** | 1 | 1 | | 1 | | |
| **M2** | | | 1 | | 1 | |
| **M3** | 1 | | | 1 | | |
| **M4** | | | 1 | | | 1 |
| **M5** | | | 1 | | | 1 |

## 3. DCA — Doğrudan Kümeleme Algoritması

### Fikir

Satır ve sütun toplamlarına göre sıralayıp 1'leri köşegene kaydır.

### Adımlar

1. Her satır ve sütundaki 1'leri say (toplamları hesapla)
2. **Satırları azalan sırada** diz (en çok 1'i olan satır yukarıya)
3. **Sütunları artan sırada** diz (en az 1'i olan sütun sola)
4. **Sütun öteleme**: İlk satırdaki 1'li sütunları sola kaydır; ikinci satır için tekrarla
5. **Satır öteleme**: En soldaki sütundaki 1'li satırları yukarı kaydır; devam et
6. Köşegen boyunca yoğun 1 bloklarını hücre olarak işaretle

!!! example "Adım Adım Örnek (Yukarıdaki Matris)"
    **Satır toplamları:** M1=3, M2=2, M3=2, M4=2, M5=2

    **Sütun toplamları:** P1=2, P2=1, P3=3, P4=2, P5=1, P6=2

    **Satır sırası (azalan):** M1 önce, geri kalanlar eşit (herhangi sıra)

    **Sütun sırası (artan):** P2(1), P5(1) önce, sonra diğerleri

    Ötelemeler sonrasında nihai matris:

    | | P3 | P5 | P6 | P1 | P2 | P4 |
    |---|:-:|:-:|:-:|:-:|:-:|:-:|
    | **M2** | 1 | 1 | | | | |
    | **M4** | 1 | | 1 | | | |
    | **M5** | 1 | | 1 | | | |
    | **M1** | | | | 1 | 1 | 1 |
    | **M3** | | | | 1 | | 1 |

!!! success "Sonuç"
    - **Hücre #1**: M2, M4, M5 → P3, P5, P6
    - **Hücre #2**: M1, M3 → P1, P2, P4
    - Blok dışı 1 yok → istisna yok

### Darboğaz Makine

Bir makine **iki ayrı hücrenin** parçalarına aynı anda gerekiyorsa → darboğaz. Köşegen bloğun dışında "yalnız" kalan 1'ler bunu gösterir.

**Çözümler:**

| Yöntem | Ne yaparsın? |
|--------|-------------|
| **Çoğaltma** | Aynı makineden bir tane daha al |
| **Fason** | O parçaları dışarıya yaptır |
| **Yeniden tasarla** | Parçanın rotasını değiştir |
| **Hücreler arası** | Makineyi iki hücre arasına yerleştir |

## 4. ROC — Rank Order Clustering

### Fikir

Her satır ve sütunu ikili sayı gibi düşün → ondalık değere çevir → büyükten küçüğe sırala.

### İkili Ağırlık

En soldaki sütun → en büyük 2'nin kuvveti.

**Örnek: 4 sütun** → ağırlıklar: $8, 4, 2, 1$ (soldan sağa)

**Satır değeri:**
$$R_i = \sum_{p=1}^{m} b_{ip} \cdot 2^{m-p}$$

**Sütun değeri:**
$$C_j = \sum_{p=1}^{n} b_{pj} \cdot 2^{n-p}$$

### Adımlar

1. Sütun ağırlıklarını yaz ($2^{m-1}, ..., 2^0$), her **satır** için ondalık değer $R_i$ hesapla
2. Satırları $R_i$'ye göre **azalan** sırala
3. Satır ağırlıklarını yaz ($2^{n-1}, ..., 2^0$), her **sütun** için ondalık değer $C_j$ hesapla
4. Sütunları $C_j$'ye göre **azalan** sırala
5. Matris değişmeyene kadar tekrarla

!!! warning "Sık Hata: Ağırlık Yönü"
    En sol/üst konumdaki en **büyük** ağırlık alır (2'nin en büyük kuvveti). Ters yazarsan ($1, 2, 4, 8$ gibi) sonuç tamamen değişir. Ağırlıkları tablonun üstüne yaz, sonra sırala.

!!! example "3×3 Örnek"
    $B = \begin{bmatrix}1&0&1\\0&1&1\\1&0&0\end{bmatrix}$ (M1-M3, P1-P3)

    **Satır değerleri** (sütun ağırlıkları sağdan sola: 4, 2, 1):

    $R_{M1} = 1(4)+0(2)+1(1) = 5$, $R_{M2} = 0+2+1 = 3$, $R_{M3} = 4+0+0 = 4$

    **Satır sırası** (azalan): M1(5) → M3(4) → M2(3)

    **Sütun değerleri** yeni sırada (satır ağırlıkları: 4, 2, 1):

    $C_{P1} = 1(4)+1(2)+0(1) = 6$, $C_{P2} = 0+0+1 = 1$, $C_{P3} = 1(4)+0+1 = 5$

    **Sütun sırası** (azalan): P1(6) → P3(5) → P2(1)

    Sonuç: sol üstte {M1,M3}×{P1,P3} bloğu oluşuyor = Hücre.

## 5. Hollier Algoritması — Hücre İçi Makine Sırası

### Amaç

Hücreler belirlendikten **sonra**: hücre içindeki makineleri hangi sıraya koymak gerekir? Hedef: ileri yönlü akışı maksimize etmek.

### Geliş-Gidiş Matrisi

Makineler arası parça hareketi sayısı. Satır = gönderen (from), sütun = alan (to).

### Formüller

$$G_i = \sum_j f_{ij} \quad \text{(makine i'nin gönderdiği parça toplamı)}$$

$$A_i = \sum_j f_{ji} \quad \text{(makine i'nin aldığı parça toplamı)}$$

$$r_i = \frac{G_i}{A_i} \quad \text{(oran — büyük = akışın başında)}$$

### Adımlar

1. Geliş-Gidiş matrisinden $G_i$ (satır toplamı) ve $A_i$ (sütun toplamı) hesapla
2. $r_i = G_i / A_i$ oran hesapla
3. Oranları **azalan** sırala → yüksek oran = akışın başında

!!! info "Özel durumlar"
    - $A_i = 0, G_i > 0$ → oran = $\infty$ → **en başa** koy (hücreye giren ham malzemenin ilk işlendiği makine)
    - Eşit oran → gelişi ($A_i$) büyük olan öne gelir

!!! example "Hollier Örneği (4 Makine)"
    | From\To | M1 | M2 | M3 | M4 | $G_i$ | $A_i$ | $r_i$ |
    |---|:-:|:-:|:-:|:-:|:-:|:-:|:-:|
    | **M1** | — | 5 | 0 | 25 | 30 | 50 | 0,60 |
    | **M2** | 30 | — | 0 | 15 | 45 | 45 | 1,00 |
    | **M3** | 10 | 40 | — | 0 | 50 | 0 | ∞ |
    | **M4** | 10 | 0 | 0 | — | 10 | 40 | 0,25 |

    **Sıra** (azalan $r_i$): M3(∞) → M2(1,00) → M1(0,60) → M4(0,25)

    | Hareket türü | Adet | Oran |
    |---|---|---|
    | İleri sıralı (komşuya) | 40+30+25=95 | %70,4 |
    | Atlamalı (ileri, aradan atlayarak) | 10+15=25 | %18,5 |
    | Geri dönüş | 5+10=15 | %11,1 |

## 6. DCA vs. ROC Karşılaştırma

| Özellik | DCA | ROC |
|---------|-----|-----|
| **Yaklaşım** | Toplamla sıralama + öteleme | İkili ağırlık + ondalık sıralama |
| **Hücre çizer mi?** | Evet, blokları işaretler | **Hayır**, yalnız sıralar |
| **Büyük matrikste** | Ağır çalışır | Her turda sistematik |
| **Sonuç** | Blok görsel olarak belirgin | Blok elle işaretlenir |

---

## 7. Sık Yapılan Hatalar

!!! warning "Dikkat"
    - **DCA'da sütunları artan, satırları azalan** — karıştırmayın.
    - **ROC hücre çizmez** — sıralama bitti, yoğun 1 bloğunu elle işaretle.
    - **Hollier'de G = satır toplamı, A = sütun toplamı** — ters almak sırayı bozar.
    - **$A_i = 0$** → bölme yapma, $r_i = \infty$ yaz, en başa koy.
    - **DCA'da öteleme sırası**: önce sütun öteleme, sonra satır öteleme — ters yapma.

## 8. Pratik Sorular

!!! question "Soru 1"
    DCA'da satırlar ve sütunlar nasıl sıralanır? Öteleme hangi sırayla yapılır?

??? success "Cevap"
    Satırlar **azalan** (en çok 1'i olan üste), sütunlar **artan** (en az 1'i olan sola). Öteleme: önce **sütun öteleme** (satır bazında), ardından **satır öteleme** (sütun bazında).

!!! question "Soru 2"
    ROC'ta bir satırın ondalık değeri neden önemlidir?

??? success "Cevap"
    ROC her satırı ikili sayı gibi okur. Sol konumdaki 1'ler büyük ağırlık → büyük ondalık değer → azalan sıralamada yukarı taşınır. Bu yolla aynı hücredeki parçalar ve makineler birbirine yakın sıraya gelir, blok oluşur.

!!! question "Soru 3"
    Hollier sıralamasında $r_i = \infty$ olan makine nereye konur? Neden?

??? success "Cevap"
    **En başa** konur. $A_i = 0$ olması, o makinenin hiçbir makineden parça almadığını gösterir — yani hücreye giren ham malzemenin ilk işlendiği makinedir, akışın başlangıç noktasıdır.
